# Contributing to This Template

Thank you for your interest in improving this template!

## How to Contribute

### Reporting Issues
- Use the issue tracker for bugs, questions, or feature requests
- Provide clear descriptions and reproduction steps
- Include relevant configuration files (sanitized)

### Suggesting Improvements
- Open an issue describing the improvement
- Explain the use case and benefits
- Provide examples if possible

### Submitting Changes
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test thoroughly with different tech stacks
5. Update documentation
6. Commit with clear messages (`git commit -m 'Add amazing feature'`)
7. Push to your branch (`git push origin feature/amazing-feature`)
8. Open a Pull Request

## Template Guidelines

### Design Principles
- **Simplicity**: Keep it simple and easy to understand
- **Flexibility**: Support multiple tech stacks and use cases
- **Security**: Follow security best practices
- **Documentation**: Clear, comprehensive documentation
- **Maintainability**: Easy to maintain and extend

### Code Standards
- Use clear, descriptive variable names
- Add comments for complex logic
- Keep configurations generic and customizable
- Provide examples for common scenarios

### Testing
Before submitting:
- [ ] Test with at least one tech stack (Node.js, Python, Go, or Rust)
- [ ] Verify local Docker build works
- [ ] Test Docker Compose setup
- [ ] Verify deployment script works (if GCP access available)
- [ ] Check documentation is updated

### Documentation
- Update README.md for major changes
- Add examples to TEMPLATE_USAGE.md
- Update inline comments in configuration files
- Keep documentation concise and practical

## Tech Stack Examples

When adding support for a new tech stack:
1. Add Dockerfile example in README.md
2. Add setup instructions in TEMPLATE_USAGE.md
3. Test the complete workflow
4. Document any special requirements

## Questions?

Feel free to open an issue for questions or clarifications.
